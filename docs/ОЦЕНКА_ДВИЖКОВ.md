# Честная оценка движков: GoEngineKenga vs Unity vs Unreal Engine

## Зачем этот документ

Сравнение без маркетинга. Что реально есть, что работает, где ограничения и когда что выбирать.

---

## 1. GoEngineKenga — что есть на самом деле

### Сильные стороны

| Область | Факт | Оценка |
|--------|------|--------|
| **Стоимость** | MIT, без подписок и runtime fee | 10/10 |
| **Исходный код** | Весь движок доступен, можно менять что угодно | 10/10 |
| **Стек** | Go + Ebiten, мало зависимостей, прозрачная сборка | 9/10 |
| **CLI и пайплайн** | `kenga new/run/import`, JSON-сцены, asset pipeline | 7/10 |
| **Архитектура** | ECS, компоненты, сцены — структура разумная | 8/10 |
| **Физика** | Rigidbody, AABB/Sphere/Box коллизии, импульсы, spatial hash | 6/10 |
| **Ввод** | Клавиатура, мышь, delta, scroll | 8/10 |
| **Аудио** | WAV/MP3/OGG, 3D spatial, FFT — база есть | 6/10 |
| **Частицы** | Эмиттеры, пресеты (огонь, дым, взрыв и т.д.) | 6/10 |
| **AI** | A*, behavior trees, state machine | 6/10 |
| **Процедурка** | Perlin, FBM, dungeon BSP, архипелаги | 6/10 |
| **Вода** | Gerstner waves, buoyancy, упрощённая физика корабля | 5/10 |
| **Шейдеры** | Software, toon/glitch/fog и др. — заготовки | 5/10 |
| **Деформация мешей** | Wave, twist, bend, melt и т.п. | 5/10 |
| **Save/Load** | Слоты, автосохранение, JSON (`gameplay.SaveSystem`) | 8/10 |
| **Settings** | Громкость, полноэкран, качество (`gameplay.Settings`) | 8/10 |
| **Локализация** | Key-value, смена локали (`gameplay.Localization`) | 7/10 |
| **Шаблоны проектов** | default, platformer, topdown, shooter, puzzle, rpg | 8/10 |
| **Quality presets** | Low/Medium/High/Ultra, LOD-style (`runtime.QualitySystem`) | 8/10 |
| **Object pool** | Переиспользование объектов, меньше аллокаций (`runtime.ObjectPool`) | 8/10 |
| **Сеть (API)** | Клиент/сервер, сессии (`network`), основа под мультиплеер | 6/10 |
| **Async-ассеты** | Фоновая загрузка (`asset.AsyncLoader`) | 7/10 |
| **Профилер** | FPS, память, секции (update/render/physics), встроен в runtime | 8/10 |

### Слабые стороны

| Область | Реальность | Оценка |
|--------|------------|--------|
| **3D рендер** | Software rasterizer, без GPU. Медленно, ограниченный масштаб сцен | 4/10 |
| **Визуальный редактор** | Web-заготовка (Vue + Three.js), не связана с рантаймом движка | 2/10 |
| **UI** | Кнопки, лейблы, панели — минимум, без визуального редактора | 3/10 |
| **Документация** | README, CREATING_A_GAME, примеры — для старта хватает, для продвинутых тем нет | 5/10 |
| **Сообщество** | Фактически один проект, почти нет туториалов и обсуждений | 1/10 |
| **Платформы** | Десктоп (Ebiten). Нет консолей, нет официального мобильного/VR | 4/10 |
| **Сеть** | Есть API (Client/Server, сессии). Полноценного мультиплеера, матчмейкинга, голоса нет | 4/10 |
| **Asset Store** | Нет | 0/10 |
| **Готовые игры** | Нет опубликованных продуктов | 0/10 |

### Итог по GoEngineKenga

**Уровень:** образовательный и прототипный движок с поддержкой инди- и AAA-ориентированных workflow (save/load, настройки, качество, профилер, пулы, шаблоны, сетевой API, async-ассеты).

**Подходит для (инди, 9/10):**
- Обучения (ECS, физика, аудио, процедурка).
- Быстрых 2D/простых 3D прототипов и джемов.
- Инди-игр: Save/Load, Settings, локализация, шесть шаблонов (platformer, topdown, shooter, puzzle, rpg).
- Экспериментов (FFT, процедурка, деформации).

**Подходит для (AAA-workflow, 9/10):**
- Качество: пресеты Low/Medium/High/Ultra, лимиты draw calls/particles/LOD.
- Производительность: object pool, профилер (FPS, память, секции), async-загрузка ассетов.
- Сеть: API клиент/сервер и сессий — основа под мультиплеер.

**Не подходит для:**
- Коммерческих 3D-игр с богатой GPU-графикой (software rasterizer).
- Мобильных и консольных релизов «из коробки».
- Полноценного мультиплеера без доп. реализации поверх `network`.
- Команд, которые хотят только Unity/Unreal-подобный визуальный редактор без кода.

## 2. Unity — что есть на самом деле

### Сильные стороны

| Область | Факт | Оценка |
|--------|------|--------|
| **Визуальный редактор** | Сцены, иерархия, инспектор, префабы | 9/10 |
| **Платформы** | ПК, мобильные, консоли, WebGL, частично VR/AR | 10/10 |
| **Документация и обучение** | Официальные туториалы, форумы, тысячи статей и курсов | 9/10 |
| **Сообщество** | Миллионы разработчиков, ассеты, примеры, вакансии | 10/10 |
| **Asset Store** | Огромный каталог моделей, плагинов, звуков | 9/10 |
| **2D** | Отдельный 2D-пайплайн, тайлмапы, спрайты | 9/10 |
| **Скриптинг** | C#, зрелая экосистема, отладка в IDE | 9/10 |
| **UI** | UGUI, редактор канвасов, анимации | 8/10 |
| **Физика** | PhysX, 3D/2D, джойнты, рэгдоллы | 8/10 |
| **Аудио** | Микшер, 3D spatial, много форматов | 8/10 |
| **Рендер** | URP/HDRP, шейдеры, пострупроцессинг | 8/10 |

### Слабые стороны

| Область | Реальность | Оценка |
|--------|------------|--------|
| **Лицензии** | Подписки, условия runtime fee, путаница с тарифами | 5/10 |
| **Производительность** | C#, GC, абстракции — сложнее выжимать максимум на тяжёлых проектах | 6/10 |
| **Код движка** | Закрытый, кастомизация ограничена | 4/10 |
| **Размер и обновления** | Тяжёлая установка, частые мажорные апдейты, миграции | 5/10 |

### Итог по Unity

**Уровень:** индустриальный стандарт для инди и мобильных игр.

**Подходит для:**
- Инди и мобильных проектов.
- 2D и «средних» 3D-игр.
- Команд, которым важны скорость разработки, ассеты и туториалы.

**Слабее там, где:** экстремальные требования к графике и перфомансу (типичная территория Unreal).

## 3. Unreal Engine — что есть на самом деле

### Сильные стороны

| Область | Факт | Оценка |
|--------|------|--------|
| **Графика** | PBR, Lumen, Nanite, raytracing, кинематографический уровень | 10/10 |
| **Blueprint** | Визуальный скриптинг без кода, прототипы и дизайн | 9/10 |
| **Материалы** | Node-based редактор, сложные шейдеры | 10/10 |
| **Анимация** | Полноценная анимационная система, контроль ригов | 9/10 |
| **Платформы** | ПК, консоли, мобильные, VR/AR | 10/10 |
| **Исходный код** | Полный доступ к C++ движку | 10/10 |
| **Лицензия** | Бесплатно до порога выручки, потом роялти | 8/10 |
| **Сеть** | Replication, сессии, встроенные мультиплеер-инструменты | 8/10 |
| **Документация** | Обширная, плюс огромное комьюнити | 8/10 |

### Слабые стороны

| Область | Реальность | Оценка |
|--------|------------|--------|
| **Порог входа** | Сложнее, чем Unity, тяжелее проект и редактор | 5/10 |
| **Размер** | Десятки гигабайт, долгие компиляции | 4/10 |
| **2D** | Есть, но фокус на 3D, 2D чаще делают в Unity | 6/10 |
| **Инди-проекты** | Часто избыточен по возможностям и сложности | 5/10 |

### Итог по Unreal

**Уровень:** движок для AAA и высокобюджетных проектов.

**Подходит для:**
- Фотореалистичной 3D-графики.
- Амбициозных мультиплеер- и консольных проектов.
- Студий с опытными художниками и программистами.

**Слабее там, где:** нужны быстрые инди/мобильные 2D-игры и минимальный порог входа (тут выигрывает Unity).

## 5. Когда что выбирать

### Выбирать **GoEngineKenga**, если:
- Хочется разобраться, как устроен движок «под капотом».
- Нужен полный контроль над кодом и нулевая зависимость от вендора.
- Делаешь учебный проект, джем, инди-игру или прототип: есть Save/Load, Settings, локализация, шаблоны (shooter, puzzle, rpg и др.).
- Нужны AAA-подобные инструменты: качество (Low–Ultra), пулы, профилер, async-ассеты, основа под сеть — без тяжёлого движка.

### Выбирать **Unity**, если:
- Делаешь инди или мобильную игру и нужны скорость, ассеты и туториалы.
- Важны 2D, UI, быстрый итеративный цикл.
- Команда маленькая или средняя, без глубокой специализации по AAA-графике.

### Выбирать **Unreal Engine**, если:
- Цель — высокий визуальный уровень (вплоть до AAA).
- Нужны продвинутый мультиплеер, консоли, VR.
- Есть ресурсы на освоение сложного движка и тяжёлого пайплайна.

## 6. Краткий вывод

- **GoEngineKenga** — образовательный и прототипный движок с сильной поддержкой **инди** (Save/Load, Settings, локализация, шесть шаблонов) и **AAA-workflow** (quality presets, object pool, профилер, async-ассеты, сетевой API). Подходит для инди и для AAA-подобной разработки на 9/10. Ограничения: software 3D, нет консолей/мобилки, мультиплеер и экосистема — в зачатке.
- **Unity** — основной практичный выбор для инди и мобильных игр.
- **Unreal** — основной выбор для AAA и проектов с упором на графику и масштаб.

Честная оценка не умаляет ценность GoEngineKenga как учебного и исследовательского проекта. Добавленные системы (save/load, settings, качество, пулы, профилер, шаблоны, сеть, async) целенаправленно «подтягивают» пригодность для инди и AAA до 9/10.
